# User Story Mapping (Jeff Patton)

Lay the work out in 2 dimensions instead of a flat backlog:
- **Backbone (horizontal)** — the user's journey: the big activities in order ("Sign up → Configure → Invite → Use → Pay").
- **Steps (under each activity)** — the tasks a user does.
- **Stories (vertical, by priority)** — under each step, ordered top = essential, lower = nice-to-have.

**Release slices** are *horizontal* cuts across the whole map — each slice is a coherent, shippable outcome
(a thin walking skeleton end-to-end), NOT "finish activity 1 then activity 2".

## Rules
- Map the journey first, features second. The backbone tells the story even to non-PMs.
- MVP = the thinnest horizontal slice that delivers the outcome end-to-end.
- Surfaces gaps (a step with no stories) and over-investment (a step with 10) at a glance.
