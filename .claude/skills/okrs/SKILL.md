# OKRs

- **Objective** — qualitative, inspiring, time-boxed direction ("Make onboarding effortless").
- **Key Results** — 3-5 *measurable outcomes* that prove the objective ("activation rate 40%→60%").
  KRs measure **outcomes**, not tasks/launches.

## Rules / anti-patterns to catch
- A KR you can mark done by *shipping something* is a task, not a result — rewrite as the metric it moves.
- Every KR needs a baseline and a target. No baseline → first KR is "establish baseline".
- 3-5 KRs per objective; if everything is a priority, nothing is.
- Set guardrails so a KR can't be hit by harming another metric.
- Grade 0.0-1.0; ~0.7 is healthy ambition, 1.0 every time means sandbagging.

Output format: Objective → KRs (baseline → target) → guardrails → check-in cadence.
